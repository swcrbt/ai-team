# AI Team

AI Team 是一个本地优先的 Flutter 桌面应用，用 Codex-like 工作台组织多模型协作。用户可以配置模型、角色、成员、团队和项目安全策略，并通过群聊或私聊完成分析、命令审批、diff 审阅和审计留痕。

## 产品边界

- 应用不依赖后端服务，状态、会话、审计和缓存由应用配置管理。
- README 不记录任何机器专属的绝对目录；持久化位置以设置页和运行平台配置为准。
- API Key 通过 secret store 抽象保存；配置导出默认不包含密钥，只有用户显式选择时才导出。
- 模型不能直接改项目文件；所有项目写入都先生成 unified diff，用户确认后才应用。
- 思考区只展示 provider 返回的 reasoning/thinking 字段，不生成或转述隐藏推理。
- 底层队列领域模型保留兼容，但当前 UI 不提供任务队列入口或队列操作控件。

## UI 结构

主侧边栏固定为 64px 深色 icon rail，入口顺序为：

```text
消息 / 团队 / 模型 / 角色 / 成员 / 项目 / 审计 / 设置
```

消息工作台只展示群聊和私聊。聊天区由顶部 header、中间可滚动消息流和底部固定 composer 组成；发送按钮固定在右下角，token 圆形进度放在发送按钮左侧，hover/focus 弹层展示上下文、输入、输出和缓存命中。

## 管理页面

- 团队：以卡片管理开发团队、测试团队、文档团队等团队对象，可编辑成员组合与协作模式，并从团队发起群聊。
- 模型：以模型列表维护 provider、模型名、Base URL、上下文窗口、流式开关、温度和最大 Token。
- 角色：以角色列表维护职责、提示词、命令策略、项目读取和补丁权限。
- 成员：以成员列表维护角色与模型绑定，并可从成员行打开私聊。
- 项目：展示项目列表、边界状态、命令审批和补丁确认。
- 审计：按最新优先展示命令、补丁、模型诊断和配置变更记录，支持过滤和详情查看。
- 设置：管理持久化存储、导入导出和应用级配置。

## 核心流程

命令审批以独立消息状态呈现：待审批可允许或拒绝，允许后可执行、停止或查看日志，执行后展示输出摘要。不同状态不会合并在同一张卡片里。

Diff 审阅以文件 tab、增删统计、展开视图和确认/拒绝操作呈现。补丁确认同时进入消息流和项目安全页，但不会塞进用户消息。

团队会话由秘书成员分配和汇总，成员输出可以在团队会话和成员私聊中查看。群聊共享上下文，私聊保持成员视角。

## 开发

架构边界见 [docs/architecture.md](docs/architecture.md)。公共导入入口保持兼容；新增代码优先放在职责更明确的 `application`、`ui`、`core/workspace`、`core/commands`、`core/orchestration` 和 `core/model` 模块。

准备依赖：

```sh
flutter --version
flutter pub get
```

运行桌面应用：

```sh
flutter run -d macos
```

构建 debug 包：

```sh
flutter build macos --debug
```

## 验证

实现类改动提交前至少运行：

```sh
flutter test
flutter analyze
flutter build macos --debug
```

如果改动影响聊天滚动、composer 或会话切换，再运行：

```sh
flutter test integration_test/chat_scroll_position_e2e_test.dart -d macos
```

文档类变更可只运行：

```sh
git diff --check
```

## 完成标准

- 消息栏只显示群聊和私聊，不出现模型、项目、工作区或任务队列行。
- 聊天 composer 固定底部，消息列表可滚动，回到底部按钮不占用消息流空间。
- 团队、模型、角色、成员、项目、审计、设置均为主侧边栏独立页面。
- 命令执行必须经过策略和审批链路。
- 项目修改必须经过 diff 确认。
- 配置可导入导出，密钥导出必须显式确认。
- 相关改动完成验证并提交到 git 后，才报告完成。
