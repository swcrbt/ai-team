# AI Team

AI Team 是一个 Codex-like Flutter 桌面应用，用来把多模型协作组织成可配置、可审批、可审计的团队工作台。它面向代码协作场景，核心体验不是任务队列，而是群聊、私聊、命令审批、Diff 审阅、项目安全和配置管理。

## 产品定位

- 用团队、模型、角色和成员定义协作关系。
- 用群聊承载团队协作，用私聊承载单个成员的上下文和审阅视角。
- 用项目页集中处理项目列表、边界状态、命令审批、补丁确认和审计摘要。
- 用审计页追踪命令、补丁、模型诊断和配置变更。
- 用设置页管理持久化存储、导入导出和应用级配置。

## UI 结构

主侧边栏是深色 icon rail，入口顺序固定为：

```text
消息 / 团队 / 模型 / 角色 / 成员 / 项目 / 审计 / 设置
```

消息工作台由会话栏、聊天区和底部 composer 组成：

- 会话栏只显示群聊和私聊，不显示模型、项目、工作区或任务队列行。
- 聊天区保持 header、可滚动消息流和固定底部 composer。
- 发送按钮固定在 composer 右下角，token 圆形进度位于发送按钮左侧。
- 回到底部按钮悬浮在 composer 上方，不占用消息流空间。
- 思考区只展示 provider 明确返回的 reasoning/thinking 内容，不生成或改写隐藏推理。

## 管理页面

- 团队管理：按开发团队、测试团队、文档团队、发布团队等团队对象做卡片式管理，并编辑成员组合、秘书成员和协作模式。
- 模型管理：按模型列表维护 provider、模型名、Base URL、上下文窗口、流式开关、温度和最大 Token。
- 角色管理：按角色列表维护职责、提示词、命令策略、项目读取和补丁权限。
- 成员管理：按成员列表维护角色与模型绑定，并可从成员行打开私聊。
- 项目管理：按项目列表展示边界状态、审批入口、补丁确认和风险摘要。
- 审计日志：按最新优先展示记录，支持过滤和详情查看。
- 设置：管理持久化目录、导入导出、密钥导出确认和应用级配置。

## 关键交互

命令请求按独立消息状态展示，不把多个状态塞进同一卡片：

- 待审批：显示允许和拒绝。
- 允许中或执行中：显示停止和日志入口。
- 已执行：展示输出摘要。
- 已拒绝或失败：展示原因。

Diff 审阅以文件 tab、增删统计、展开视图和确认/拒绝操作呈现。补丁确认可以出现在消息流和项目页，但不能放进用户消息里。

群聊天然共享团队上下文，不提供额外的“共享群聊上下文”开关。私聊保留成员身份、相关文件和审阅状态。

## 数据与安全

- API Key 通过 secret store 抽象保存；脱敏导出默认不包含密钥。
- 持久化目录由设置页管理，README 不记录开发者机器上的绝对路径。
- 命令执行必须经过策略检查和用户审批。
- 项目修改必须先形成 Diff，并在用户确认后应用。

## 开发

更详细的架构边界见 [docs/architecture.md](docs/architecture.md)。公共导入入口保持兼容；新增代码应跟随现有职责边界，不为单次需求引入额外抽象。

准备依赖：

```sh
flutter --version
flutter pub get
```

运行桌面应用：

```sh
flutter run -d macos
```

构建 debug 包：

```sh
flutter build macos --debug
```

## 验证

实现类改动提交前至少运行：

```sh
flutter test
flutter analyze
flutter build macos --debug
```

如果改动影响聊天滚动、composer 或会话切换，再运行：

```sh
flutter test integration_test/chat_scroll_position_e2e_test.dart -d macos
```

文档类变更可只运行：

```sh
git diff --check
```

## 完成标准

- 消息栏只显示群聊和私聊，不出现模型、项目或任务队列行。
- 聊天 composer 固定底部，消息列表可滚动，回到底部按钮不占用消息流空间。
- 团队、模型、角色、成员、项目、审计和设置均为主侧边栏独立页面。
- 管理页采用列表优先、选中后编辑详情的结构。
- 命令执行必须经过策略和审批链路。
- 项目修改必须经过 Diff 确认。
- 配置可导入导出，密钥导出必须显式确认。
- 相关改动完成验证并提交到 git 后，才报告完成。
